System overview — how the pieces talk

Three layers behind stable interfaces. The React frontend renders a chatbot; the FastAPI gateway (server/app.py) wraps the Phase-1 ReAct engine (data_agent_baseline/) and streams its real trace over SSE. Tools and LLM endpoints are plug-and-play.

Live run — a question, end to end

What happens when the user asks a real data question in Live mode. The engine runs in a worker thread; events stream back over SSE. In Co-pilot the engine pauses at on_propose() and waits for a decision posted to /api/decide.

ReAct engine loop — inside ReActAgent.run()

The tuned reasoning loop in agents/react.py. Each iteration: build messages → call the model → parse → guard checks → (co-pilot gate) → execute tool → stream + diagnose → repeat until answer. Guards keep the agent from looping or submitting malformed answers.

Frontend data-flow — events become UI

The hook useAgentRunLive.send() is the only place that talks to the backend. Greetings are answered by the LLM; questions stream a run. Incoming AgentEvents map to feed / results / plan, which the panels render. Mock and live hooks share the exact same shape, so components never change.

Tools & providers — plug-and-play

ToolRegistry.execute(action, input) dispatches to a handler; each handler delegates to a focused module. New tools are added via the UI (ToolsModal) or a YAML entry — no core change. LLM and data endpoints sit behind adapters so they swap freely.

Event contract & the two modes

One normalized stream is the single source of truth between backend and UI (lib/eventContract.js). The same events render the trace in both modes — the only difference is whether the engine pauses for you.
